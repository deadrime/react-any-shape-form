# Hooks

React Any Shape Form provides a set of powerful hooks for fine-grained control over form state. These hooks enable you to access and manipulate form data from anywhere in your component tree.

## Overview

All hooks are attached to the form instance created by `useForm` or `createForm`:

```tsx
const MyForm = useForm({ name: "", age: 0 });

// Access hooks via MyForm
const name = MyForm.useWatch("name");
const [age, setAge] = MyForm.useField("age");
const { errors, validationStatus } = MyForm.useFieldValidation("name");
```

## Hooks Reference

### [useWatch](/docs/documentation-hooks-usewatch--docs)

Subscribe to a field's value and re-render when it changes.

```tsx
const value = MyForm.useWatch(fieldName);
```

### [useField](/docs/documentation-hooks-usefield--docs)

Get a field's value and a setter function for updating it.

```tsx
const [value, setValue] = MyForm.useField(fieldName);
```

### [useFieldValidation](/docs/documentation-hooks-usefieldvalidation--docs)

Get validation state and errors for a specific field.

```tsx
const { errors, validationStatus } = MyForm.useFieldValidation(fieldName);
```

### [useArrayField](/docs/documentation-hooks-usearrayfield--docs)

Manage array fields with add, remove, and update operations. Available via `withArrayFields()` addon.

```tsx
const { value, append, remove, update } = MyForm.useArrayField(fieldName);
```

### [useFormState](/docs/documentation-hooks-useformstate--docs)

Get a reactive snapshot of the overall form state. Available via `withFormState()` addon.

```tsx
const { isDirty, dirtyFields, touchedFields, isSubmitting, isValid } = MyForm.useFormState();
```

### [useFieldState](/docs/documentation-hooks-usefieldstate--docs)

Get per-field state: touched, dirty, and validity. Re-renders only when that specific field's state changes. Available via `withFormState()` addon.

```tsx
const { isTouched, isDirty, isValid } = MyForm.useFieldState(fieldName);
```

## Combining Hooks

You can use multiple hooks together for complex scenarios:

### Example: Conditional Field

Show a field only when another field has a specific value:

```tsx
const MyForm = useForm({
  accountType: "personal",
  companyName: ""
});

function ConditionalCompanyField() {
  const accountType = MyForm.useWatch("accountType");

  if (accountType !== "business") {
    return null;
  }

  return (
    <MyForm.Item name="companyName">
      {({ value, onChange }) => (
        <input
          placeholder="Company Name"
          value={value}
          onChange={(e) => onChange(e.target.value)}
        />
      )}
    </MyForm.Item>
  );
}
```

### Example: Live Character Counter

```tsx
const MyForm = useForm({
  bio: ""
});

function BioField() {
  const bio = MyForm.useWatch("bio");
  const maxLength = 200;
  const remaining = maxLength - bio.length;

  return (
    <div>
      <MyForm.Item name="bio">
        {({ value, onChange }) => (
          <textarea
            value={value}
            onChange={(e) => onChange(e.target.value)}
            maxLength={maxLength}
          />
        )}
      </MyForm.Item>
      <div className={remaining < 20 ? "warning" : ""}>
        {remaining} characters remaining
      </div>
    </div>
  );
}
```

### Example: Dependent Validation

Validate one field based on another:

```tsx
const MyForm = useForm({
  password: "",
  confirmPassword: ""
});

function PasswordFields() {
  const password = MyForm.useWatch("password");

  return (
    <>
      <MyForm.Item
        name="password"
        rules={[
          { required: true, message: "Password is required" },
          { min: 8, message: "Minimum 8 characters" }
        ]}
      >
        {({ value, onChange, errors }) => (
          <div>
            <input
              type="password"
              value={value}
              onChange={(e) => onChange(e.target.value)}
            />
            {errors[0] && <span>{errors[0].errorText}</span>}
          </div>
        )}
      </MyForm.Item>

      <MyForm.Item
        name="confirmPassword"
        rules={[
          { required: true, message: "Please confirm password" },
          {
            validator: async (value) => {
              if (value !== password) {
                return Promise.reject("Passwords do not match");
              }
            },
            message: "Passwords must match"
          }
        ]}
      >
        {({ value, onChange, errors }) => (
          <div>
            <input
              type="password"
              value={value}
              onChange={(e) => onChange(e.target.value)}
            />
            {errors[0] && <span>{errors[0].errorText}</span>}
          </div>
        )}
      </MyForm.Item>
    </>
  );
}
```

## Custom Hooks

Build your own hooks on top of the built-in ones:

### Example: useFormField (Combined Hook)

```tsx
function useFormField<T>(
  form: ReturnType<typeof useForm<any>>,
  fieldName: string
) {
  const [value, setValue] = form.useField(fieldName);
  const { errors, validationStatus } = form.useFieldValidation(fieldName);

  return {
    value,
    setValue,
    errors,
    validationStatus,
    hasError: errors.length > 0,
    isValidating: validationStatus === "validating"
  };
}

// Usage
function MyComponent() {
  const MyForm = useForm({ email: "" });
  const email = useFormField(MyForm, "email");

  return (
    <div>
      <input
        value={email.value}
        onChange={(e) => email.setValue(e.target.value)}
      />
      {email.isValidating && <span>Validating...</span>}
      {email.hasError && <span>{email.errors[0].errorText}</span>}
    </div>
  );
}
```

## Hook Best Practices

### 1. Use the Right Hook for the Job

- **useWatch** - When you only need to read a value
- **useField** - When you need to read and update a value
- **useFieldValidation** - When you need validation info
- **useArrayField** - When working with arrays

### 2. Minimize Re-renders

Use `useWatch` instead of accessing the entire form state:

```tsx
// ✗ Bad - causes re-renders on any field change
function EmailDisplay() {
  const state = MyForm.formApi.getState();
  return <div>{state.email}</div>;
}

// ✓ Good - only re-renders when email changes
function EmailDisplay() {
  const email = MyForm.useWatch("email");
  return <div>{email}</div>;
}
```

### 3. Extract Complex Logic

Move complex field logic into separate components:

```tsx
// Extract complex field into its own component
function PasswordField() {
  const [password, setPassword] = MyForm.useField("password");
  const { errors, validationStatus } = MyForm.useFieldValidation("password");
  const [showPassword, setShowPassword] = useState(false);

  return (
    <div>
      <input
        type={showPassword ? "text" : "password"}
        value={password}
        onChange={(e) => setPassword(e.target.value)}
      />
      <button onClick={() => setShowPassword(!showPassword)}>
        {showPassword ? "Hide" : "Show"}
      </button>
      {/* ... error display, strength meter, etc ... */}
    </div>
  );
}

// Use in main form
function MyForm() {
  return (
    <Form onSubmit={handleSubmit}>
      <PasswordField />
    </Form>
  );
}
```

### 4. Use Type Safety

Let TypeScript guide you:

```tsx
const MyForm = useForm({
  age: 0,
  name: "",
  tags: [] as string[]
});

// TypeScript ensures you pass the right field name
const age = MyForm.useWatch("age");     // ✓ Valid
const invalid = MyForm.useWatch("xyz"); // ✗ Error

// TypeScript ensures array operations only on array fields
const { fields } = MyForm.useArrayField("tags");  // ✓ Valid
const { fields } = MyForm.useArrayField("name");  // ✗ Error (not an array)
```

### 5. Avoid Stale Closures

When using hooks in callbacks, ensure you have the latest values:

```tsx
function MyComponent() {
  const [count, setCount] = MyForm.useField("count");

  // ✗ Bad - count might be stale in async callback
  const handleClick = async () => {
    await someAsyncOperation();
    setCount(count + 1); // count might be outdated
  };

  // ✓ Good - use functional update
  const handleClick = async () => {
    await someAsyncOperation();
    const currentCount = MyForm.formApi.getFieldValue("count");
    setCount(currentCount + 1);
  };

  return <button onClick={handleClick}>Increment</button>;
}
```
