### Key Concepts

In this example:
- **`useForm`** creates a form instance with initial state
- **`MyForm.Item`** connects a field to the form state
- **Render prop pattern** gives you full control over input rendering
- **You handle all styling** - the library provides zero CSS

The `children` render function receives:
- `value` - current field value
- `onChange` - function to update the value
- `errors` - array of validation errors
- `validationStatus` - validation state: `'notStarted' | 'validating' | 'success' | 'error'`

## Validation

React Any Shape Form includes powerful, promise-based validation:

### Built-in Rules

The library provides common validators out of the box:


```tsx
<MyForm.Item
  name="email"
  rules={[
    { required: true, message: "Email is required" },
    { type: "email", message: "Invalid email format" },
    { min: 5, message: "At least 5 characters" },
    { max: 100, message: "Maximum 100 characters" }
  ]}
>
  {({ value, onChange, errors }) => (
    // Your input component
  )}
</MyForm.Item>
```

### Custom Validators

Create async validators for complex logic:

```tsx
{
  validator: async (value) => {
    const exists = await checkUsernameExists(value);
    if (exists) {
      return Promise.reject("Username taken");
    }
  },
  message: "Username already exists",
  validateTrigger: ["onChange", "onSubmit"]
}
```

### Validation Triggers

Control when validation runs:
- **`onChange`** - validates as user types (debounced)
- **`onSubmit`** - validates on form submission

See the [Validation](/docs/documentation-validation--docs) section for detailed validation documentation.

## Creating Forms: `useForm` vs `createForm`

React Any Shape Form provides two ways to create forms, depending on your needs:

### Option 1: `useForm` - For Simple Forms

Use `useForm` when building a simple form contained within a single component:

```tsx
import { useForm } from 'react-any-shape-form';

function MyComponent() {
  const MyForm = useForm({
    name: "",
    email: ""
  });

  return (
    <MyForm onSubmit={(state) => console.log(state)}>
      <MyForm.Item name="name">
        {({ value, onChange }) => (
          <input value={value} onChange={(e) => onChange(e.target.value)} />
        )}
      </MyForm.Item>
      {/* ... more fields ... */}
    </MyForm>
  );
}
```

**Characteristics:**
- Created inside a React component
- Scoped to that component's lifecycle
- Perfect for single-component forms
- Form instance is recreated on component remount

### Option 2: `createForm` - For Shared Forms

Use `createForm` when you need to:
- Share form state across multiple components
- Access form outside React components
- Persist form instance across remounts
- Build reusable form definitions

```tsx
import { createForm } from 'react-any-shape-form';

// Created outside component - shared across your app
const LoginForm = createForm({
  username: "",
  password: ""
});

// Use in multiple places
function LoginPage() {
  return (
    <LoginForm onSubmit={(state) => handleLogin(state)}>
      <LoginForm.Item name="username">
        {({ value, onChange }) => (
          <input value={value} onChange={(e) => onChange(e.target.value)} />
        )}
      </LoginForm.Item>
      {/* ... */}
    </LoginForm>
  );
}

// Access form state in another component
function LoginStatus() {
  const username = LoginForm.useWatch("username");
  return <div>Logging in as: {username}</div>;
}

// Or access imperatively outside React
LoginForm.formApi.setFieldValue("username", "admin");
const currentState = LoginForm.formApi.getState();
```

**Characteristics:**
- Created outside React components
- Single instance shared across your app
- Can be accessed anywhere via `FormApi`
- Form state persists across component remounts
- Ideal for complex, multi-step, or shared forms

### When to Use Which?

- **Simple form in one component** — `useForm`
- **Multi-step wizard** — `createForm`
- **Form shared across routes** — `createForm`
- **Access form outside React** — `createForm`
- **Persist state on unmount** — `createForm`
- **Quick prototype** — `useForm`

## Form API Access

Every form provides a `formApi` instance for programmatic control:

<Canvas of={FormApiStories.UsingFormApi} />

### Common FormApi Methods

```tsx
// Get/set field values
MyForm.formApi.getFieldValue("name");
MyForm.formApi.setFieldValue("name", "John");
MyForm.formApi.setFieldsValue({ name: "John", age: 30 });

// Get entire form state
const state = MyForm.formApi.getState();

// Validation
await MyForm.formApi.validateFields(); // validate all fields
await MyForm.formApi.validateFields(["email"]); // validate specific fields

// Form submission
await MyForm.formApi.submit(); // validates and calls onSubmit

// Reset
MyForm.formApi.resetFields(); // reset to initial state
MyForm.formApi.resetFields(["email"]); // reset specific fields
```
